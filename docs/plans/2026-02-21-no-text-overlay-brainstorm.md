# KV 无字生图 + 前端叠字 Brainstorm

## 1. 问题定义

当前链路把营销文字直接交给生图模型渲染，导致两类问题：

1. 中文可读性不稳定（乱码、错字、字形畸变）。
2. 文案排版不可控（字号、位置、层级、按钮样式不一致）。

项目定位是简历演示项目，不追求生产级复杂链路，但要在演示中稳定输出“可读中文 + 统一品牌排版”。

---

## 2. 可选方案

### 方案 A：继续模型内渲染文字（优化 Prompt）

做法：
1. 增加“文字清晰、短句排版”提示。
2. 强化负面词约束（乱码、错字）。
3. 增加多候选再选优。

优点：
1. 改动小，开发快。
2. 无需改前端合成逻辑。

缺点：
1. 中文质量仍然是概率问题。
2. 不能保证排版一致。

结论：适合短期缓解，不适合“稳定演示质量”目标。

### 方案 B：无字生图 + 前端 Canvas 叠字（推荐）

做法：
1. 生成阶段强约束“只画图不写营销文案”。
2. 每张海报生成 `overlaySpec`（标题/副标题/卖点/CTA/位置/颜色）。
3. 在前端用 Canvas 合成最终带字海报。

优点：
1. 中文文字 100% 可控。
2. 排版系统化，品牌一致性高。
3. 适合做作品集“AI 原图 vs 成品图”对比展示。

缺点：
1. 需要新增前端合成层。
2. 需处理不同画幅下的适配。

结论：最符合当前目标。

### 方案 C：服务端叠字（API/Edge）

做法：
1. 后端接收原图 + overlaySpec 合成输出。
2. 前端只展示最终图。

优点：
1. 终端一致性更高。
2. 可扩展为生产化架构。

缺点：
1. 工程复杂度高。
2. 不符合本轮“演示优先、快速迭代”目标。

结论：本轮不采用。

---

## 3. 推荐方案与边界

推荐采用方案 B，并设定本轮边界：

1. 先覆盖海报 01/02/09 三个核心类型（Hero/Lifestyle/Specs）。
2. 先做基础叠字能力：标题、副标题、3条卖点、CTA。
3. 输出保留两张图：
   - `rawUrl`：AI 无字原图
   - `url`（final）：前端叠字成品图
4. 画廊默认展示成品图；“查看详情”支持切换看原图与成品图。

---

## 4. 验收标准（脑爆阶段）

1. 不再依赖模型渲染中文营销文案。
2. 同一套输入重复生成时，中文排版样式稳定一致。
3. 画廊可对比原图/成品图，便于演示讲解。
4. 本地可部署并完整走通“上传 -> 生成 -> 预览 -> 下载”。
