# 简历优化器升级设计文档

> **项目类型:** 渐进式升级 (小步快跑)
> **设计日期:** 2026-02-19
> **目标:** 优化 AI 分析质量和 UI/UX 体验

---

## 一、项目概述

### 1.1 背景

现有简历优化器已实现基础功能(JD+简历输入、Kimi AI 分析、Markdown 结果展示),但存在以下问题:

- **AI 分析质量不足:** 输出结构不清晰,缺少量化指标,建议优先级不明确
- **视觉体验一般:** 色彩单调,布局不够优化,缺少现代感
- **交互体验粗糙:** 缺少流畅的动画和反馈,用户感知不佳

### 1.2 升级目标

本次升级采用**渐进式方案**,重点优化:

1. **AI 分析质量:** 重构 prompt 模板,提供结构化、可操作的分析结果
2. **视觉设计:** 采用 Claude 风格配色,简洁优雅的界面
3. **交互体验:** 添加流畅的动画和微交互,提升用户体验

### 1.3 设计原则

- ✅ **YAGNI:** 只实现必要的功能,避免过度设计
- ✅ **快速迭代:** 小步快跑,快速验证效果
- ✅ **保持兼容:** 不破坏现有功能,架构保持稳定
- ✅ **用户体验:** 每个优化都应有明显的用户价值

## 二、AI 分析质量优化

### 2.1 当前问题

现有 prompt 模板输出的问题:
- 缺少量化指标(只有文字描述)
- 输出结构不够清晰
- 建议缺少优先级排序
- 关键词优化不够具体

### 2.2 优化方案

#### 2.2.1 新的分析维度体系

```
1. 匹配度评分 (量化指标)
   - 总体匹配度: X/100
   - 核心能力匹配: X/100
   - 经验相关性: X/100
   - 技能覆盖度: X/100

2. 关键发现 (结构化列表)
   - ✅ 3-5 个高匹配亮点 (标注"必写")
   - ⚠️ 3-5 个关键缺口 (标注"必补")
   - 💡 差异化优势点 (标注"加分项")

3. 分级优化建议
   - 🔴 高优先级 (3个): 直接影响录用,必须修改
   - 🟡 中优先级 (3个): 提升竞争力,建议修改
   - 🟢 低优先级 (3个): 锦上添花,可选修改

4. 简历重写示例
   - 自我评价重写 (150字)
   - 项目经历重写 (2个关键项目)
   - 技能清单优化

5. 关键词优化清单
   - JD 高频关键词提取 (TOP 10)
   - 建议融入位置 (具体到哪个模块)
```

#### 2.2.2 Prompt 工程技巧

**使用的技巧:**
- ✅ **CoT (Chain of Thought):** 让 AI 展示思考过程
- ✅ **Few-shot Learning:** 提供优秀分析示例
- ✅ **结构化约束:** 明确输出格式和结构
- ✅ **量化要求:** 要求具体数字和评分
- ✅ **优先级排序:** 明确建议的优先级

**新 prompt 特点:**
- 更严格的输出格式约束
- 要求 AI 给出具体的评分和数字
- 要求标注每个建议的优先级
- 要求提供可操作的重写示例

### 2.3 输出格式优化

#### 2.3.1 新增结构化元素

**评分卡片:**
- 总体匹配度: 大号数字 + 进度条
- 分项评分: 4 个维度的小型进度条
- 颜色编码: 红/橙/绿 对应 低/中/高匹配度

**标签系统:**
- 高匹配: 绿色标签 + ✅ 图标
- 待优化: 橙色标签 + ⚠️ 图标
- 加分项: 蓝色标签 + 💡 图标

**可操作建议清单:**
- 带优先级标记 (🔴🟡🟢)
- 可勾选的复选框
- 折叠/展开详细说明

**对比表格:**
- 简历 vs JD 的技能对比
- 缺失技能高亮显示

## 四、动画和微交互

### 4.1 页面级动画

#### 4.1.1 页面加载

```css
入场动画序列 (总时长 600ms):
1. Header: fade-in + slide-down (0-200ms)
2. 输入框: stagger fade-in + slide-up (200-400ms, 延迟 50ms)
3. 按钮: scale-in + fade-in (400-600ms)
4. 结果区: 如有内容, fade-in (600-700ms)

缓动函数: cubic-bezier(0.16, 1, 0.3, 1) - 自然弹性
```

#### 4.1.2 滚动行为

```css
平滑滚动:
  - scroll-behavior: smooth
  - 滚动时元素视差效果 (轻微)
  - 结果区域进入视口时触发淡入
```

### 4.2 交互动画

#### 4.2.1 按钮微交互

```css
分析按钮状态机:
1. 默认状态:
   - 背景: #D97757

2. 悬停状态 (150ms):
   - 背景变深: #C26647

3. 点击状态:
   - 瞬间收缩: scale(0.96)

   - 图标替换为 Spinner
   - 文字: "分析中..."
   - 背景色: #9A9A9A
   - 禁用指针事件

5. 完成状态 (400ms 庆祝):
   - ✅ 图标短暂显示
```

#### 4.2.2 输入框交互

```css
焦点动画:
1. 边框过渡: 200ms ease-out
   - 从 #E8E8E6 → #D97757
   - 外发光: 0 0 0 3px rgba(217,119,87,0.1)

2. 字符计数:
   - 接近限制时变色: 绿 → 橙 → 红
   - 轻轻的抖动提示 (超过限制时)

3. 快捷按钮:
   - 悬停: 图标旋转 180° (200ms)
   - 点击: 涟漪效果
```

### 4.3 结果展示动画

#### 4.3.1 内容渐进式展现

```css
分析结果加载序列 (总时长 1.2s):

1. 评分卡片 (0-300ms):
   - 进度条从 0% → 实际分数
   - 数字滚动动画 (countUp)
   - 颜色根据分数渐变

2. 匹配度标签 (300-600ms):
   - Stagger 方式逐个出现
   - 每个延迟 50ms
   - 从左侧滑入 + fade-in

3. 详细分析 (600-900ms):
   - 整体 fade-in + slide-up
   - Markdown 内容逐行渲染

4. 建议清单 (900-1200ms):
   - 列表项逐个展开
   - 每项带复选框交互
```

#### 4.3.2 标签页切换

```css
切换动画 (300ms):
1. 当前内容: fade-out + scale(0.98) (0-150ms)
2. 指示器滑动: translateX (0-300ms, easeInOut)
3. 新内容: fade-in + scale(1) (150-300ms)
```

#### 4.3.3 评分卡片动画

```css
进度条:
  - 初始宽度: 0%
  - 目标宽度: 实际分数%
  - 持续时间: 800ms
  - 缓动: cubic-bezier(0.16, 1, 0.3, 1)
  - 颜色渐变:
    < 60%: #C53030 (红)
    60-80%: #D97706 (橙)
    > 80%: #5B8C5A (绿)

数字滚动:
  - 从 0 开始
  - 每帧 +1-2 (根据总分)
  - 总时长: 600ms
  - ease-out 缓动
```

#### 4.4.1 图标动画

```css
Sparkles (分析按钮):
  - 旋转: 360° / 3s

  - 悬停: 上下跳动 (2px, 2次)
```

```css

  - 尺寸: 24px

  - 动画: rotate 360° / 800ms
  - 线条粗细: 3px
  - 阴影: 轻微发光

骨架屏 (分析结果):
  - 脉冲动画: opacity 1 → 0.6 → 1
  - 周期: 1.5s
  - 渐变背景: linear-gradient(90deg,
      #F3F3F2 0%, #F9F9F8 50%, #F3F3F2 100%)
```

#### 4.4.3 反馈动画

```css
成功反馈:
  - ✅ Checkmark 出现
  - 圆形背景扩散 (ripple)
  - 持续 600ms
  - 颜色: #5B8C5A

错误反馈:
  - ❌ 图标轻微摇晃 (shake)
  - 红色边框闪烁 2 次
  - 持续 400ms
  - 颜色: #C53030

复制成功:
  - 📋 图标短暂显示

  - 滑出 (200ms)
```

### 4.5 性能优化

```css
动画最佳实践:
1. 使用 transform 和 opacity (GPU 加速)
2. 避免动画 width/height (触发 reflow)
3. 使用 will-change 提示浏览器
4. 减少动画的阴影和模糊 (移动端)

示例:
  .animate-slide {
    will-change: transform, opacity;
    transform: translateZ(0); /* 强制 GPU */
  }

媒体查询:
  - 移动端: 减少动画时长 (50%)
  - 低端设备: 完全禁用动画
  - 用户偏好: respects prefers-reduced-motion
```
