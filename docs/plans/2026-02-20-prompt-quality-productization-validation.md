# KV Prompt Quality Productization Validation

## 1. 验证目标

验证本轮改造后的四项能力是否生效：

1. 执行轨 Prompt（runtimePromptEn/runtimeNegative）已接入生图调用。
2. 生图请求已携带参考图（上传图）作为输入。
3. 质量模式（fast/balanced/quality）能控制候选数量。
4. 每张海报使用候选选优后输出最佳结果。

---

## 2. 静态验证

执行：

```bash
npm run lint
npm run typecheck
```

预期：

1. ESLint 通过。
2. TypeScript 编译通过。

---

## 3. 功能路径验证（手工）

### 3.1 基础流程

1. 启动：`npm run dev`
2. 上传产品图 -> 分析 -> 风格页 -> 提示词页。
3. 在提示词页切换质量模式并选择海报类型。
4. 点击开始生成，观察生成页状态和文案。

### 3.2 质量模式验证

分别测试：

1. `fast`：每张仅 1 候选。
2. `balanced`：每张 2 候选。
3. `quality`：每张 3 候选。

预期：生成页文案出现 `候选 x/y`，且 y 与模式匹配。

### 3.3 参考图锁定验证

选海报01（Hero）并重复两次生成，观察：

1. 包装轮廓是否保持一致。
2. LOGO 拼写与位置是否稳定。
3. 主色调与图标区域是否明显跑偏。

预期：相较旧版，错 Logo/错包装比例明显下降。

---

## 4. A/B 对比建议

对同一产品图做 2 组实验：

1. A 组（旧链路）：使用 `promptEn + negative` 单次生成。
2. B 组（新链路）：使用 `runtimePromptEn + runtimeNegative` + 候选选优。

对比维度（每项 1~5 分）：

1. 包装还原度
2. 品牌一致性
3. 构图可用性
4. 视觉美感
5. 文本可读性

建议至少记录海报 01 / 02 / 09 三张。

---

## 5. 回归检查清单

1. 提示词页复制展示 Prompt 功能正常。
2. 仍支持单张/多张/全部输出。
3. 比例选择结果仍正确传到生成接口。
4. 生成失败时错误提示正常。

---

## 6. 已知限制（本轮）

1. 候选选优采用轻量规则，不是视觉模型打分。
2. 中文长文案渲染仍可能不稳定（后续建议前端后处理覆盖）。
3. 尚未实现候选图片在画廊页切换展示（当前仅保留最优图）。
