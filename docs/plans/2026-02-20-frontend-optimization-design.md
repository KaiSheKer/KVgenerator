## 1. 参考图识别

### 1.1 布局结构识别

参考图是典型的「控制台式双栏工作台」：

1. 左侧窄栏（控制面板）
- 顶部上传区域
- 中段参数配置（产品信息、场景规划）
- 底部风格与语言设置
- 主操作按钮（生成）

2. 右侧宽栏（结果工作区）
- 顶部大预览区（主海报 + 进度条）
- 中部结果网格（多张海报卡片）
- 底部主次操作按钮（继续生成 / 修改配置）

### 1.2 风格识别

1. 视觉基调
- 深色背景（近黑蓝）
- 面板半透明 + 细边框 + 内发光
- 高对比的霓虹强调色（蓝紫 / 品红）

2. 组件风格
- 大圆角卡片（20-24px）
- 渐变主按钮（蓝紫到洋红）
- 低饱和灰文本 + 高亮标题
- 卡片 hover 上浮 + 阴影增强

3. 交互特征
- 清晰分层：输入在左、结果在右
- 实时状态可视化（进度条 + 状态卡片）
- 结果卡片具备直接下载能力

### 1.3 元素清单

1. 必备元素
- 上传卡片
- 输入表单区
- 风格/语言配置块
- 主预览画布
- 结果缩略图网格
- 渐变 CTA 按钮

2. 状态元素
- 生成中/成功/失败状态
- 进度条
- 当前任务提示

## 2. 目标设计（落地到本项目）

### 2.1 信息架构目标

保持现有多步骤业务流程（上传 → 分析 → 编辑 → 风格 → 提示词 → 生成 → 画廊），但统一为同一套视觉语言：

1. 全局统一深色霓虹主题
2. 主要页面采用「控制区 + 内容区」布局语义
3. CTA 与状态组件样式统一

### 2.2 视觉 Token

1. 背景层
- `--color-background`: 深蓝黑
- 背景叠加径向渐变营造氛围

2. 表面层
- `--color-card` / `--color-surface`: 深色半透明
- `--color-border`: 低对比细边框

3. 品牌层
- `--color-primary`: 电蓝紫
- `--color-accent`: 品红紫
- 主按钮统一 gradient

4. 语义层
- success / warning / error 用于进度状态

### 2.3 组件规则

1. Card
- 默认 20px+ 圆角、细边框、柔和阴影

2. Button
- `default`: 渐变 + 发光阴影
- `outline`: 深色描边 + 轻 hover

3. Input
- 深色半透明底 + 明确 focus ring

4. Progress
- 深色轨道 + 渐变填充

### 2.4 页面级设计

1. 首页
- 左侧上传与流程提示，右侧预览占位区
- 上传后主按钮显著突出

2. 风格/提示词页面
- 使用大卡片与统一标题层级
- 选中态用发光 ring 和轻微 scale

3. 生成页面
- 大进度条 + 状态卡片矩阵
- 对失败状态保留可读错误反馈

4. 画廊页面
- 结果卡片统一比例、支持悬浮操作
- 下载行为与文案保持一致

## 3. 非功能目标

1. 响应式
- 1366+ 双栏布局
- 平板/手机自动折叠为单栏

2. 可访问性
- 所有按钮可键盘触达
- 文本与背景保持足够对比

3. 性能
- 结果图统一使用 `next/image`
- 避免不必要重渲染和无限 effect

---

## 4. 验收标准

1. 视觉一致性
- 主色、圆角、阴影、交互反馈风格统一

2. 流程可用性
- 全流程无阻断（上传到下载）

3. 可靠性
- `npm run test` 通过（lint + typecheck）
