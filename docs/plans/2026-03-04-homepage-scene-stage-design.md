# 2026-03-04 Homepage Auto Banner Design

## Goal

优化首页首屏结构，在不改动左侧上传与流程模块的前提下，把右侧区域改造成固定高度的纯图片自动轮播区：

- 未上传时：展示自动轮播的作品 banner
- 上传后：切换为用户图片预览区

## Scope

本次改造只覆盖首页 `/`。

- 保持左侧现有模块和交互不变
- 保持两栏布局，不新增第三列或额外内容区
- 优先保证桌面端首屏观感

## Confirmed Decisions

### 1. 页面只保留两个区域

- 左侧：上传模块 + 流程概览
- 右侧：图片轮播区 / 预览区

### 2. 左侧模块不改

左侧现有内容保持不变：

- 上传产品图片
- 流程概览
- 开始分析按钮

允许做的仅有一项：为了和右侧高度对齐，给左侧容器加固定高度适配能力。

### 3. 右侧未上传状态

右侧未上传状态只展示图片，不展示任何额外 UI：

- 不要场景切换按钮
- 不要箭头
- 不要分页点
- 不要进度条
- 不要标题与说明文字

右侧每一帧保留一组两张图，并按固定时间自动切换到下一组：

- 左侧：原图
- 右侧：设计图
- 两张图处于同一个统一展示面内，而不是两个独立卡片

### 4. 右侧上传后状态

当用户上传图片后，右侧整块区域切换为预览区：

- 停止轮播
- 移除 banner 内容
- 只保留大图预览主体

### 5. 轮播方式

轮播为最简单实现：

- 固定时间切图
- 直接切换
- 不做前后图叠层
- 不做渐变过渡

## Layout Design

### Desktop

首页在桌面端采用固定舞台高度：

- 页面内容区高度基于视口高度计算
- 左右两栏共享同一高度
- 右侧 Card 内部只保留一个完整图片舞台

推荐结构：

- 根布局：`xl:grid-cols-[320px_minmax(0,1fr)]`
- 整体高度：`calc(100vh - header - footer - shell padding)`
- 左侧：纵向两块，上传区域撑满剩余高度
- 右侧：一个完整的全高舞台

### Mobile / Narrow Viewports

移动端不强制固定首屏高度，保留顺序降级：

- 左侧模块先显示
- 右侧舞台后显示

## Right Stage Design

### Unuploaded State

结构极简：

- 一个全高舞台区域
- 内部固定展示当前轮播组的两张图
- 两张图左右并排，但共享同一个整体外框

图片要求：

- 尽量占据右侧大部分高度
- 展示内容为两张独立图片
- 左侧为原图，右侧为设计图
- 页面不负责解释图片内容

### Uploaded State

上传后，右侧切换为预览区：

- 使用同一块全高舞台
- 展示用户上传图的大预览
- 保持舞台尺寸不跳变

## Data Strategy

右侧未上传态支持两种图片来源：

1. 用户自备 banner 分组数组
2. 当前仓库内置的占位视觉图（兜底）

这样在没有真实图片时，首页仍然可运行；当用户把真实图片放到指定目录后，只需更新分组数组即可替换为真实轮播图。

## Acceptance Criteria

### Functional

- 左侧模块内容与操作不变
- 未上传时右侧自动轮播原图/设计图分组
- 上传图片后右侧切换为预览区域
- 轮播在图片数量为 0 或 1 时不报错

### Visual

- 桌面端左右两栏高度一致
- 右侧图片展示区域明显大于当前版本
- 页面不出现按钮、控件、说明文案

- `npm run build` 通过
