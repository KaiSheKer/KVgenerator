# 2026-02-23 Prompt Edit + Poster Refine Design

## 1. 背景与问题
当前流程是：上传 -> 分析 -> 风格 -> 提示词 -> 生成 -> 画廊。
存在两个核心体验缺口：

1. 提示词页面的英文 Prompt 仅展示，无法直接编辑；用户无法在生成前做细粒度定向微调。
2. 画廊页面只能查看/下载，无法对已生成结果发起二次优化；用户需要回到前序步骤重跑，效率低。

本次改造目标是把产品打磨成“演示级可控生成器”：支持在生成前编辑 Prompt，并在生成后基于单张图持续迭代。

## 2. 目标与非目标
### 目标
1. 在提示词页支持每张海报 `promptEn` 的可编辑、可复制、可恢复模板。
2. 在画廊详情弹窗中提供“二次优化输入框”，仅作用于当前海报。
3. 支持单图版本迭代（v1/v2/v3），并能在详情弹窗中切换查看。

### 非目标
1. 不做多图批量优化。
2. 不做对话式长链路优化助手。
3. 不引入后端持久化历史（会话级状态即可）。

## 3. 交互设计
### 3.1 提示词页
1. 英文 Prompt 区改为可编辑文本框。
2. 增加“复制英文 Prompt”按钮。
3. 增加“恢复英文模板”按钮。
4. 点击“开始生成”时，把编辑后的 Prompt 快照写入生成数据，确保生成使用用户编辑值。

### 3.2 画廊详情弹窗
1. 维持现有大图预览、缩放、拖拽能力。
2. 增加版本切换按钮（v1/v2/...）。
3. 增加底部优化区（参考图示）：
   - 海报编号与标题
   - “提示词详情”折叠区
   - 输入优化意见文本框
   - 提交优化按钮
4. 提交后仅重绘当前海报，并追加新版本。

## 4. 数据与状态设计
### 4.1 Prompt 编辑状态
在提示词页维护 `editablePromptEnById: Record<string, string>`，按 `poster.id` 存储编辑值。

### 4.2 海报版本状态
扩展 `GeneratedPoster`：
- `versions?: GeneratedPosterVersion[]`
- `activeVersionId?: string`

新增 `GeneratedPosterVersion`：
- `id: string`（如 v1/v2）
- `url: string`
- `source: 'initial' | 'refine'`
- `note?: string`
- `createdAt: number`

## 5. 生成策略
### 5.1 初次生成
每张海报生成成功后写入 `v1`，并设置 `activeVersionId='v1'`。

### 5.2 二次优化
在详情弹窗提交优化时：
1. 读取当前海报可见版本图作为参考图。
2. 将 `base promptEn + refinement instruction + continuity constraint` 拼接为最终 Prompt。
3. 调用现有生成 API。
4. 成功后追加新版本并切到最新版本；失败时保留旧版本。

## 6. 异常与边界
1. 未输入优化意见：前端阻止提交并提示。
2. 当前图不可用：提示无法优化。
3. API 限流/失败：显示错误，不覆盖当前版本。
4. 历史无 versions 的兼容对象：自动视作单个 `v1`。

## 7. 验证标准
1. 英文 Prompt 编辑后参与实际生成。
2. 单图优化成功后出现新版本且可切换。
3. 失败重试不破坏当前版本。
4. 类型检查通过，主流程可回归运行。
